# Разработка

## Вход в код

- **Страницы и layout**: `app/`, вложенность повторяет URL.
- **Проект и вкладки**: `app/app/p/[id]/layout.tsx`, `ProjectTabs.tsx`, отдельные `page.tsx` под каждую вкладку.
- **Состояние и API**: `lib/store.ts` + соответствующие `route.ts` рядом с фичами.
- **Таймлайн**: `timeline/TimelineViewer.tsx` (большой компонент: сетка, drag, стили).
- **Печать**: `export/print/PrintWorkspaceClient.tsx`, стили печати в `app/globals.css`, масштаб в `lib/print-timeline-scale.ts`.

## Проверка типов

```bash
npx tsc --noEmit
```

## Локальные данные

При разработке создаётся **`data/`** в корне репозитория. Его можно удалить для «чистого» стенда (потеряются пользователи и проекты). В git каталог обычно не коммитят — при необходимости добавьте `data/` в `.gitignore`.

## Сессия в dev

Без `SB_SECRET` в `lib/session.ts` используется запасной секрет для разработки. Для приближения к продакшену задайте `SB_SECRET` в `.env.local` (Next подхватывает переменные из `.env*` для серверного кода).

## Добавление фич

- Новые серверные эндпоинты — как **`export const dynamic`** не требуется по умолчанию; следите за вызовами `cookies()` и сессией только на сервере.
- Клиентские эффекты печати держите в компонентах с `"use client"`, чтобы не смешивать с RSC без нужды.
